**Bridgic** 是一个创新的编程框架，旨在创建代理系统，从确定性工作流到自主代理。它引入了一种新的范式，简化了代理系统的开发。

## 核心特性

* **编排**: Bridgic 通过利用预定义的依赖关系和动态路由来帮助管理您的 AI 应用程序的执行流程。
* **参数绑定**: 有三种方法可以在 Worker 之间传递数据——参数映射、参数注入和输入传播——从而消除全局状态管理的复杂性。
* **动态路由**: Bridgic 通过易于使用的 `ferry_to()` API 实现条件分支和智能决策，适应运行时动态。
* **动态拓扑**: 在 Bridgic 中，拓扑可以在运行时更改，以支持高度自主的 AI 应用程序。
* **模块化**: 在 Bridgic 中，可以通过层次嵌套重用组件来构建复杂的代理系统。
* **人在回路**: Bridgic 风格的代理系统可以在需要时请求人类反馈，以动态调整其执行逻辑。
* **序列化**: Bridgic 采用可扩展的序列化和反序列化机制，实现状态持久化和恢复，使长时间运行的 AI 系统能够实现人在回路。
* **系统集成**: 各种工具和 LLM 可以以系统化的方式无缝集成到 Bridgic 世界中。
* **定制化**: Bridgic 提供的不是一种“黑箱”方法。您可以完全控制 AI 应用程序的各个方面，例如提示、上下文窗口、控制流等。
