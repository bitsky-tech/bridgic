# 可观测性

## 为什么可观测性很重要

开发和管理自主系统面临着自身的挑战。与传统软件的可预测执行流程不同，自主系统涉及动态决策、复杂工作流和与外部服务的交互。如果没有适当的可观测性，这些系统可能会变成“黑箱”——你知道输入和输出，但对中间发生的事情几乎没有洞察。

有效的可观测性提供了几个关键好处：

- **理解运行时行为**：查看系统执行时实际发生的事情，包括哪些任务正在运行、它们的执行顺序以及它们之间的相互作用。

- **任务级可见性**：跟踪单个任务单元的输入和输出，使你能够理解数据流，识别转换发生的位置，并在适当的细节级别调试问题。

- **性能优化**：帮助测量执行时间并识别使用 Bridgic 构建的自主系统的性能瓶颈。这种可见性对于持续优化自主系统至关重要。

## 可观测性如何工作

### Worker 执行

在 Bridgic 框架中，每个任务执行发生在 **Worker** 粒度。Worker 是表示可以执行的离散任务的基本单元。这种以 Worker 为中心的架构为可观测性仪器提供了自然的边界，无论复杂性如何。无论你是在构建简单的顺序工作流还是复杂的基于图的自主系统（如 ReAct），每个操作都作为一个 Worker 运行。

### 回调机制

Bridgic 提供了一个灵活的回调机制，可以在关键生命周期点挂钩到 Worker 执行中。这个回调系统设计为非侵入式——回调观察和仪器化执行，而不修改你的核心业务逻辑。有关此机制如何详细工作的更多信息，请参阅 [Worker 回调机制](../core_mechanism/worker_callback.md)。
